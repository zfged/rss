Клиент состоит из  трех страниц:
admin.html - страница где пользователель с правами админ может редактировать ленты (добавлять удалять изменять)
item.html - страница где пользователь и админ могут видеть перечень лент и разпарсить содержимое конкретной ленты
index.html страниа где пользователь или админ могут авторизоваться и пользователь зарегистрироваться

Cервер. Все контролеры построены по принципу rest архитектуры
Контроллеры:
RSSController - контроллер для редактирования лент
ItemController - контроллер для вывода конкретной ленты
AuthorizeController - контроллер для получения статуса авторизации и роли
AссountController - стандартный контроллер для полного осуществления системы получения прав
SignController - контроллер для редактирования подписок
Модели:
RSS - модель для редактирования и валидации ленты
RSSItem - модель для получения конкретной ленты (xml)
RSSContext - контекст для доступа к локальной базе данных
UserRSS - модель для проверки подписки пользователя на конкретную новость
